ReplayKit

The Rust workspace now contains the base architecture: canonical run/span types, in-memory and SQLite storage backends, a collector, a replay engine, a diff engine, an API service, and SDK helpers for semantic instrumentation.

The next frontend step is to replace this placeholder with a real local web app that reads the API and renders the run list, trace tree, span inspector, branch flow, and diff views.

  • Core model: shared contracts for runs, spans, artifacts, snapshots, edges, branches, and diffs.
  • Replay engine: dirty-set planning, branch execution, and blocked replay semantics.
  • API layer: one typed boundary for the eventual UI and CLI.

Run cargo run -p replaykit-cli -- demo to exercise the current base architecture.
